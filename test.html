Abstractâ€” Sorting is a technique to rearrange a given list of elements according to a comparison operator on the elements. There are a large number of Sorting Algorithms like Insertion Sort, Merge Sort, Bucket Sort, Shell Sort, etc.

The efficiency of the sorting algorithm depends on many factors such as memory usage patterns (the number of times the sections of memory must be copied or swapped to and from the disk), the total number of comparisons and the time requirements for the algorithms to run.

In this paper, a method is proposed which combines two sorting algorithms (Bucket Sort and Shell Sort) in a way that takes advantage of the strength of each to improve overall performance.

Keywordsâ€” Shell Sort, Bucket Sort, Insertion Sort, Sorting, time complexity, algorithm comparison

  1. Introduction

    When a list of numbers is given, special algorithms are used to arrange these numbers in order. These algorithms are called sorting algorithms. These numbers may be alike, different, in a pattern or completely random. The most frequently used orders are numerical and lexicographical orders. Sorting is a crucial step used in various algorithms. Hence, the need to optimize it is essential.

    There have been many attempts made to analyze the complexity of sorting algorithms [1] which led to the invention of various interesting and efficient algorithms as well as improvements in the already existing ones. To derive new and improved algorithms like the one discussed in this paper, it is required to study and understand the existing algorithms. The following algorithms have been used to compare the performance of the proposed one.

    Insertion Sort [2], [3] is a simple algorithm which iterates, checking one input element at once, and growing the sorted output list. It repeats until no input elements remain. Its average case time complexity is Quadratic in nature; O(n2). java.util.Arrays.sort() is an inbuilt method used to sort a list in java. It uses a variation of Quicksort (dual-pivot Quicksort) to

    sort the numbers in a list. The average time complexity for this algorithm is of the order O(n log(n)).

    Merge Sort [4], [5] divides input list into two halves and then merges the two sorted halves. Its average time complexity is of the order O(n log(n)).

    Bucket Sort [6], [7], [8] is a non-comparison sorting algorithm in which buckets distributed over a range, are created. Now, all elements are taken and are sorted, consecutively, by putting each one of them in their corresponding bucket.

    Algorithm: (Let n is the size of the list)

    1. Create buckets to hold the elements according to their magnitudes.

    2. For every list element, insert it into the bucket.

    3. Apply Insertion Sort to sort each bucket.

    4. Concatenate all the sorted buckets.


    Bucket Sort is mainly preferable when the input is uniformly distributed over a range. It is also very effective when an optimal number of buckets are used, so as to best complement the sort. One of the advantages of bucket sort is that it runs in linear time in the average case hence generally, its time requirements are low; as low as O(n+k) where k is a parameter. Bucket sort also has some limitations. To know the number of buckets required we should be aware of the value of the maximum element in the input list. Another limitation is that if all the elements enter in one bucket then it transforms into a normal insertion sort which reduces its efficiency.

    Shell Sort [9], [10] is mainly a variation of Insertion Sort where we allow the swapping of non-continuous numbers in a list. In Shell Sort, we make the given list gap-sorted for a large value of gap and keep reducing its value until it becomes 1.

    Algorithm:

    1. Initialize the value of gap.

    2. Divide the list into smaller lists of equal interval of gap.

    3. Sort these sub-lists at the interval of gap.

    4. Repeat until complete list is sorted.

    One of the main advantages of Shell Sort is that it is efficient for small lists and their repetitive sorting which would be of great significance when arranging elements of the bucket. Also, it is manifolds faster than Bubble Sort and Insertion Sort. Although its time complexity depends on the value of the gap sequence obtained during the algorithmâ€™s execution, Shell Sort generally needs O(n log(n)) time to sort a given list.

    Shell Sortâ€™s major limitation is that it is complex to understand. Another disadvantage is that its running time is heavily dependent on the gap sequence it uses. It is not as efficient as Merge Sort or Quicksort but it can provide improvements over Insertion Sort.

    The improvements provided by Shell Sort are clearly explained in the subsequent section on the proposed algorithm.

  2. Proposed Work

    As mentioned in the previous section, the proposed algorithm is an improvement over the original Bucket Sort algorithm which is used to sort the numbers of a list by using buckets. The buckets define magnitudes (range) of the elements which they can hold. After the elements are put into their corresponding buckets, Insertion sort is used to arrange them in the correct order. In this (proposed) algorithm, Shell Sort is used instead of Insertion Sort to arrange the elements inside the bucket as it generally requires lesser time.

  4. Conclusion

    As seen in the previous section, the proposed algorithm has performed poorly in a list with all elements already in sorted order. However, it has conquered various limitations faced by the corresponding traditional algorithms. It should be noted that the following conclusions have been drawn by inferring Chart 2 from the previous section.

    1. Limitations Surpassed

      1. There is no need to give the maximum value in the input list in the proposed algorithm.

      2. The proposed algorithm overcomes the limitation of unevenly distributed numbers in the list for Bucket Sort.

      3. If a list contains same number occurring multiple times, Shell Sort would take more time than the proposed algorithm.

    2. Explanation

      The original Bucket Sort algorithm uses Insertion Sort internally, to sort the elements of a bucket but this improved method, uses Shell Sort to do the same.

      When two elements in a bucket are to be sorted, a swap occurs. The swap occurs in a non-contiguous segment and moves the item over a greater distance within the overall list. Insertion Sort, on the other hand, only moves the item one position at a time. This means that in Shell Sort, the items being swapped are more likely to be closer to its final position then Insertion Sort. Since the items are more likely to be closer to its final position, the list itself become partially sorted. Also, when the gap (as in gap between the two elements to be compared) equals 1, Shell Sort is performing basically like Insertion Sort. This generally happens when most of the sorting process has been already carried out, leaving the list partially sorted. In this case, Shell Sort will be able to work very fast, since Insertion Sort is fast when the list is almost in order.

      1. The improved algorithm has an inbuilt function which calculates the maximum value of the given list. Hence there is no need to give the extra parameter.

      2. As seen from the chart, the numbers of the list are no longer needed to be evenly distributed. This is because of the use of Shell Sort instead of Insertion Sort.

      3. Bucket Sort works best with lists containing more repetitions. This is done by putting all the numbers which occur multiple times, into one bucket. This leads to less time being consumed as initially, shell sort had to compute the positions of the repeated numbers in the whole list whereas in this case, the positions need to be computed only in the given bucket.

    3. Persisting Limitations

      While the proposed algorithm combines the positive qualities of both Bucket Sort and Shell Sort, it still has some limitations that could not be overcome.

      1. The proposed algorithm is not capable of sorting a list with negative numbers. This owes to the fact the all the buckets created have a range of positive numbers.

        The current algorithm can be modified to handle negative numbers as well. The range of buckets can be shifted to negative numbers allowing negative numbers to be sorted in the manner similar to positive numbers.

      2. As mentioned above, the proposed algorithm has performed poorly in a list with all elements already in sorted order. This is because even at its best, the complexity of the algorithm is of the order O(n log(n)). The current algorithm can be modified to use other sorting techniques while sorting a list with all (or most) the elements already in place. This will reduce the time needs of the proposed algorithm but would need some prerequisite knowledge of the list to be sorted.

  5. Motivation and Need for this Paper

    This section discusses the needs and motivation of the research paper. Also, the new findings of the research so conducted have been elaborated.

    As mentioned above the original Bucket Sort algorithm uses Insertion Sort internally whereas, this algorithm uses Shell Sort. The major advantage of Shell Sort is that it efficiently sorts an unevenly distributed list which has been a vital problem in the original algorithm.

    Moreover, using Bucket Sort and Shell Sort together has overcome some problems encountered by the latter as well. This

    includes the difficulty faced by Shell Sort while rearranging a number occurring multiple times in the corresponding list. This problem was easily overcome by Bucket Sort by putting the repetitive element in the same bucket, therefore making it easier to sort.

    The motivation to write this paper is the willingness to enhance the conventional Bucket Sort algorithm and make it use the available resources more efficiently.
